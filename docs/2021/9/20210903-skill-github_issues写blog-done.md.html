如何使用Github issue 写博客

1、初衷

有很多的博客平台，方便定制、简洁明了，模板多多。但是要使用图片的话，需要用到图床（自行Bing）。文章写好后，还需要自行部署（虽然可以想办法自动化）。评论系统的对接也有些许麻烦。其他平台例如：石墨文档、ONENOTE、Evernote等，体验不是很好，或者是需要长时间适应。

2、优点

  • 可随时记录。
  • 图片直接上传
  • Github为静态服务
  • 支持markdown
  • issue自带评论系统
  • labels标签类似于同类的TAG
  • projects项目类似于同类的分类（在每个项目里还可细分）
  • issues列表类似于同类的归档

3、开始部署

  1. new repository-新建仓库，填写repository name与description。

  2. new projects-新建项目，填写project board name 与description。

  1. add column-增加专栏，填写column name。在projects里，可以添加专栏，用于分类每个issues。

  2. new issue-新建问题，在issues栏中new issues，填写Title与leave a comment，可直接插图片，右侧栏目assignees-撰写人、labels-标签、projects-所属项目。

  3. new labels-新建标签，可在issues栏中的lables位置创建新的标签。可将原来所属标签删除，new labels，填写label name、description、color。

  4. add cards-将issues加入对于projects的column中。在对应的projects中选择add cards，将对应的issues拖入对应的column即可。

  5. pin issue-将该issue固定在issues栏主页。在某一个issues具体内容中的右边栏目可pin。
