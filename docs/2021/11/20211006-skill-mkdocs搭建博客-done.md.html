MkDocs Blog教程

1、初衷

写博客的平台那么多，之前用学着用 Github issues 写博客，够用是够用，但是需要跳转链接。还不能同步到 Gitess ，导致国内访问速度很慢。无意间了解到了 Mkdocs 写博客。相当于静态界面，可部署到很多平台，且和 Github issues 写博客差不多，不需要其他的技能，只需要会用 MD 语法写文章即可。

2、开始部署

2.1 Linux平台

sudo apt install mkdocs

PS：几乎所有的文章都是用 pip 安装，但是我的 pip 安装代理有问题，倒腾了很长时间，在腾讯服务器上提示我可以用 apt 安装，我瞬间开心了起来。

2.2 新建本地项目

mkdocs new [dir-name]

新建一个项目文件夹，其目录结构如下：

mkdocs.yml    # The configuration file. 配置文件
docs/
    index.md  # The documentation homepage. 首页
    ...       # Other markdown pages, images and other files.

2.3 启动实时调试服务

mkdocs serve

进行本地调试，可修改，实时预览。[http://127.0.0.1:8000/];

2.4 换 Material 的主题

pip install mkdocs-material # failed

PS：依旧是 pip 代理问题，无法安装，所以直接将其 Git 仓库克隆了下来。

git clone git@github.com:squidfunk/mkdocs-material.git # 使用的ssh方法

PS：建议把主题包文件与 mkdocs new 的项目文件放在一起。

切换通过 Git 下来的 Material 主题，在项目文件下修改配置文件 - mkdocs.yml，加入以下代码。

theme:
  name: null
  custom_dir: mkdocs-material/material

  # 404 page
  static_templates:
    - 404.html

  # Necessary for search to work properly
  include_search_page: false
  search_index_only: true

  # Default values, taken from mkdocs_theme.yml
  language: en
  font:
    text: Roboto
    code: Roboto Mono
  favicon: assets/favicon.png
  icon:
    logo: logo

PS：If you cloned Material for MkDocs from GitHub, you must list all of the themes’ defaults, because mkdocs_theme.yml is not loaded automatically as described in the official documentation. 如果是通过 Git 下来的主题，则需要保持主题配置默认。

使用 2.3 进行预览。

2.4 发布到 Github Pages 或 GitLab Pages

mkdocs build # 构建文档站点

会在项目文件中生成一个 site 文件夹，将其上传到 Github Pages 或其他站点即可。
